- **The crux and how it was cracked.** Needed: a torus point `P : Tᵈ` with all coords ≠ 0, all inner
  fract-args ∉ ℤ, and `dGpdTorus P` strictly outside the digit window. The old `realizeR` forced

  defect target `τ ∈ (0,1)` from `exists_scale_outside_window_strict`; build `realizeR0 α c τ σ` with a
  **free seed `σ`** for `r 0` and higher coords `= Int.fract(τ + α^{k+1}σ − …)`. Key insight:
  `orbitF α c (realizeR0 α c τ σ) k = τ` **independent of `σ`** (`orbitF_realizeR0`) — the inner arg is
  `τ` shifted by an integer `−⌊X⌋`, so its fract is `τ`. Hence `dGpdTorus = τ·S_d` (outside window) and
  every inner arg is non-integer because `τ ≠ 0`. The seed `σ` is chosen in the **uncountable** `(0,1)`
  (`Cardinal.Real.Ioo_countable_iff` — note: that lemma lives in `namespace Cardinal`, so the fully
  qualified name has the `Cardinal.` prefix even though it's about `Real`) avoiding the **countable** bad
  set (each `{σ | X_k(σ) ∈ ℤ}` = an affine-preimage of `ℤ`, a `Set.range` over `ℤ`, countable).
- **The condition-2 shortcut.** The "inner args ∉ ℤ" requirement reduces to `orbitF = τ ≠ 0` (since
  `orbitF k = Int.fract(orbitArg k)`, and `fract a = 0 ⟺ a = ⌊a⌋`). So no separate `orbitArg_congr`
  lemma was needed — just `orbitF_congr` + `orbitF_realizeR0`.
- **Why `_strict` was needed.** `ae_dStep_fails_of_exceeding` wants the strict disjunction
  `< C−2 ∨ C <`; the pre-existing `exists_scale_outside_window` only gave `∉ Ioc` (i.e. `≤ C−2`). Added
  `exists_scale_outside_window_strict` (3-way case split on where `C` sits relative to `0`/`S`).
- **Base-`g` is the natural next frontier, and it's mechanical, not a wall.** The engine is nearly
  base-agnostic already: the `2` in `dStep_defect_identity` (`v_d = 2u+C−D`) comes *only* from `α^d = 2`.
  I generalized it (`dStep_defect_identity_base`, conclusion `g·u+C−D`, base-2 is the `g=2` instance) and

  now its `g=2` instance). **Caveat that's part of the base-`g` statement**: the escape bound
  `g^{1/d} < 2g/(g+1)` needs **larger `d` for larger `g`** (`d=3` works for `g=2`, fails for `g=10`), so
